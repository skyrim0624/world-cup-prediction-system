# 模拟引擎开发记录

日期：2026-06-14

## 本次目标

把世界杯预测系统从前端 mock 原型推进到可运行的动态预测模型 MVP：能计算未开赛单场胜平负、比分概率、已完赛赛果锁定、整届路径传导、48 队赛制骨架、新闻事件权重和五大盘面解释。

## 已完成内容

- 新增 FastAPI 后端，提供 `/api/match-prediction` 和 `/api/model-status`。
- 新增本地 JSON 数据层：球队、赛程、事件、来源权重、当前比赛。
- 实现 Poisson 单场比分概率、胜平负概率和最可能比分 Top 3。
- 实现蒙特卡洛路径模拟，输出 32 强、16 强、8 强、4 强、决赛和冠军概率。
- 自动补齐 48 个参赛槽位、12 个小组和 72 场小组赛结构。
- 已结束比赛锁定进积分表，不再作为预测对象。
- 实现最佳小组第三出线逻辑：12 个第三名中选 8 个。
- 接入 FIFA Regulations 里的固定淘汰赛路径：M73-M88、M89-M96、M97-M100、M101-M102。
- 接入 FIFA Regulations Annexe C 的 495 种第三名组合。
- 实现事件来源权重：低可信传闻不进模型，可信事件按权重修正底层评分。
- 实现 `modelMeta.factorImpacts`，把事件影响输出给前端解释层。
- 球队前端展示改为五大盘面：实力盘、状态盘、路径盘、人员盘、边际盘。
- 前端接入真实模型接口，支持 48 队冠军概率榜和占位球队队码 fallback。
- 增加每日预测快照生成脚本，可用 `npm run update:snapshot` 生成本地 JSON 快照。
- `/api/match-prediction` 默认优先读取每日预测快照，缺快照时回退现场计算。
- 前端交互请求使用 1,200 次模拟，正式快照/API 仍支持最高 50,000 次模拟。
- 新增 15 秒 API 短缓存，同一刷新窗口内避免重复重算。
- 新增 `npm run validate:data`，用于真实数据替换前校验球队、分组、赛程、Annex C 和事件引用。
- 新增新闻来源和原始新闻数据层：`news-sources.json`、`raw-news.json`。
- 原始新闻可推断为事件因子、影响方向和影响强度，并标记为入模、待审核或忽略。
- `/api/events` 暴露事件列表、来源等级、审核动作和摘要。

## 关键决策

- 第一版仍使用公开可维护数据和自建模型，不接未授权商业数据。
- 页面表达继续避免“赔率”语境，只展示比分概率和概率变化。
- 交互页面用较低模拟次数保证响应；正式每日更新用高模拟次数生成快照。
- D 级来源权重为 0，只作为风险线索展示，不直接进入概率模型。
- C 级来源默认不直接入模，但多源确认或人工确认后可以作为低权重半硬信息进入模型。
- 网页审核写入后，通过“重建快照”即可让正式预测读取新结果，不再需要重启 API。
- 新新闻录入只进入审核队列，不绕过来源评级、人工审核和快照重建流程。
- 已结束比赛不再预测，必须通过赛果锁定进入积分、路径和后续模拟。
- 首页现在承载当前重点单场 + 多场未开赛列表；更深的比分分布和路径传导应拆到单场详情页。
- 预测入口链路已经成立：未开赛列表 → 单场详情 → 比分分布 → 整届概率传导。
- 真实赛事数据替换必须走导入器：先完整校验，再备份，再写入；失败不得污染现有数据。
- C 级单源线索默认待审；多源交叉验证后可自动进入低权重模型流。
- 新闻抓取入口先支持公开 RSS / Atom，不绑定未授权商业 API。
- 日更流程先做成 CLI，可被 cron、GitHub Actions 或后续后台按钮调用；不先绑定具体调度平台。
- 日更状态是运行状态，不进入 Git；运营后台只读取最近一次成功状态，不替代正式调度和失败告警。
- 进行中比赛不做强实时承诺，但允许手动或外部抓取器写入 live score，让当前比分进入后续模拟。
- `uvicorn --reload` 会显著拖慢计算接口，因此默认 `npm run dev:api` 使用稳定服务模式。
