# 模拟引擎开发记录

日期：2026-06-14

## 本次目标

把前端 mock 原型推进到“真模型 MVP”：至少能用本地数据跑出单场胜平负、比分概率、路径传导和冠军概率。

## 已完成内容

- 新增 `backend/data.py`：球队评分、赛程、已结束赛果、结构化事件和来源权重。
- 新增 `backend/model.py`：Poisson 单场比分概率、事件修正、已结束比赛锁定、蒙特卡洛路径模拟。
- 新增 `backend/main.py`：FastAPI 接口 `/api/match-prediction`。
- 新增 `requirements.txt`：FastAPI 和 Uvicorn 依赖。
- 新增 `tests/test_model.py`：模型基本测试。
- 更新 `vite.config.ts`：开发环境下 `/api` 代理到 `127.0.0.1:8000`。
- 更新前端：读取接口返回的 `teams` 和 `modelMeta`，显示真实模型、模拟次数和锁定赛果。

## 关键决策

- 第一版不接外部商业数据，先用自建评分和本地事件样例。
- 第一版模拟次数设为 8,000 次，保证开发联调速度；正式更新再提高。
- D 级社媒传闻权重为 0，只展示为风险线索，不进入模型。
- 已结束比赛直接锁定进积分表，不再预测。

## 未解决问题

- 还不是完整 48 队 2026 世界杯结构。
- 还没有新闻自动抓取和多源交叉验证。
- 还没有后台录入事件。
- 还没有支付和权限。

## 下一步

- 扩完整赛程和球队数据结构。
- 把事件数据从代码常量移到可维护 JSON 或后台。
- 单场详情页展示比分分布和路径传导细节。

## 迭代记录：模拟次数配置化

日期：2026-06-14

### 本次目标

把模拟次数从代码固定值改成接口可配置，支持开发快跑和正式更新两种使用方式。

- 增加 API 测试，验证 `?simulations=1200` 生效。

### 验证结果

- `npm run test:model` 通过。
- `npm run build` 通过。
- 本地 50,000 次模拟成功返回。

### 当前判断

开发预览继续用 8,000 次；正式每日更新可以使用 50,000 次。后续如果数据扩大到 48 队，需要继续做缓存或后台任务，避免前台请求等待太久。

## 迭代记录：每日预测快照

## 迭代记录：48 队赛制骨架与 32 强路径

日期：2026-06-14

### 本次目标

把模型从 8 队样例推进到 2026 世界杯核心赛制骨架：48 队、12 组、最佳第三出线和 32 强淘汰路径。

### 已完成内容

- `backend/data.py` 自动补齐 48 个参赛槽位。
- 每组自动补足到 4 队。
- 自动生成每组 6 场小组赛，共 72 场小组赛结构。
- 新增 `group_names`。
- 新增 `best_third_place_teams`。
- `simulate_tournament` 输出 `roundOf32`、`roundOf16`、`quarterfinal`、`semifinal`、`final`、`champion`。
- 比分采样改为累计分布二分采样，减少模拟耗时。
- 测试从默认 8,000 次改为 1,200 次，保证反馈速度。

### 验证结果

- `npm run test:model` 通过，13 个测试。
- `npm run build` 通过。
- 1,200 次完整预测约 2.5 秒。

### 当前判断

赛制骨架已经接近客户要的“动态预测模型”底层，但仍不能声称完全真实。下一步要继续补官方 32 强路径映射、小组第三分支表，以及真实名单/赛程导入机制。
