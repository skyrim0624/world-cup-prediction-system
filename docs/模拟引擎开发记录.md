# 模拟引擎开发记录

日期：2026-06-14

## 本次目标

把前端 mock 原型推进到“真模型 MVP”：至少能用本地数据跑出单场胜平负、比分概率、路径传导和冠军概率。

## 已完成内容

- 新增 `backend/data.py`：球队评分、赛程、已结束赛果、结构化事件和来源权重。
- 新增 `backend/model.py`：Poisson 单场比分概率、事件修正、已结束比赛锁定、蒙特卡洛路径模拟。
- 新增 `backend/main.py`：FastAPI 接口 `/api/match-prediction`。
- 新增 `requirements.txt`：FastAPI 和 Uvicorn 依赖。
- 新增 `tests/test_model.py`：模型基本测试。
- 更新 `vite.config.ts`：开发环境下 `/api` 代理到 `127.0.0.1:8000`。
- 更新前端：读取接口返回的 `teams` 和 `modelMeta`，显示真实模型、模拟次数和锁定赛果。

## 关键决策

- 第一版不接外部商业数据，先用自建评分和本地事件样例。
- 第一版模拟次数设为 8,000 次，保证开发联调速度；正式更新再提高。
- D 级社媒传闻权重为 0，只展示为风险线索，不进入模型。
- 已结束比赛直接锁定进积分表，不再预测。

## 未解决问题

- 还不是完整 48 队 2026 世界杯结构。
- 还没有新闻自动抓取和多源交叉验证。
- 还没有后台录入事件。
- 还没有支付和权限。

## 下一步

- 把事件数据从代码常量移到可维护 JSON 或后台。
- 单场详情页展示比分分布和路径传导细节。

## 迭代记录：数据 JSON 化与 8 队样例

日期：2026-06-14

### 本次目标

把模型从“代码里写死的 4 队样例”推进到更可维护的本地数据结构，并修正多组情况下的小组排名逻辑。

### 已完成内容

- 新增 `backend/data_files/teams.json`。
- 新增 `backend/data_files/fixtures.json`。
- 新增 `backend/data_files/events.json`。
- 新增 `backend/data_files/source-weights.json`。
- 新增 `backend/data_files/current-match.json`。
- `backend/data.py` 改为从 JSON 加载数据，并做基础校验。
- 球队池扩展为 8 队，两组样例：E 组和 F 组。
- 修正 `rank_group`：只对指定小组内的球队排序。
- `simulate_tournament` 改为按组出线后再进行跨组淘汰路径模拟。
- `/api/model-status` 暴露数据覆盖范围和已知缺口。
- 前端展示事件透明度：入模事件和忽略事件数量。

### 验证结果

- `npm run test:model` 通过。
- `npm run build` 通过。
- 桌面 1280px 和手机 390px 联调无横向溢出。
- 页面显示真实模型、8 队数据和事件透明度。

### 当前判断

这一版已经比“演示模型”更接近真实产品底座：数据可维护、接口透明、分组逻辑正确。但还不是完整世界杯模型，下一步必须继续扩真实赛制和数据运营能力。
