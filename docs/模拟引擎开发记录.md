# 模拟引擎开发记录

日期：2026-06-14

## 本次目标

把前端 mock 原型推进到“真模型 MVP”：至少能用本地数据跑出单场胜平负、比分概率、路径传导和冠军概率。

## 已完成内容

- 新增 `backend/data.py`：球队评分、赛程、已结束赛果、结构化事件和来源权重。
- 新增 `backend/model.py`：Poisson 单场比分概率、事件修正、已结束比赛锁定、蒙特卡洛路径模拟。
- 新增 `backend/main.py`：FastAPI 接口 `/api/match-prediction`。
- 新增 `requirements.txt`：FastAPI 和 Uvicorn 依赖。
- 新增 `tests/test_model.py`：模型基本测试。
- 更新 `vite.config.ts`：开发环境下 `/api` 代理到 `127.0.0.1:8000`。
- 更新前端：读取接口返回的 `teams` 和 `modelMeta`，显示真实模型、模拟次数和锁定赛果。

## 关键决策

- 第一版不接外部商业数据，先用自建评分和本地事件样例。
- 第一版模拟次数设为 8,000 次，保证开发联调速度；正式更新再提高。
- D 级社媒传闻权重为 0，只展示为风险线索，不进入模型。
- 已结束比赛直接锁定进积分表，不再预测。

## 未解决问题

- 还不是完整 48 队 2026 世界杯结构。
- 还没有新闻自动抓取和多源交叉验证。
- 还没有后台录入事件。
- 还没有支付和权限。

## 下一步

- 把事件数据从代码常量移到可维护 JSON 或后台。

## 迭代记录：官方淘汰赛路径与接口短缓存

日期：2026-06-14

### 本次目标

把 32 强后的路径从 MVP 排序配对改成 FIFA Regulations 里的固定 match number 路径，并解决前端多标签/轮询重复触发模拟导致的响应变慢问题。

### 已完成内容

- 根据 FIFA World Cup 26 Regulations 2026 年 5 月版 Article 12.6-12.11，加入 M73-M88、M89-M96、M97-M100、M101-M102 的固定路径。
- `simulate_tournament` 不再把 32 强球队按排名排序后两两相邻配对，而是按官方 match number 逐轮推进。
- 新增 `build_round_of_32_matches`，用小组第一/第二和最佳第三名生成 32 强对阵。
- 新增第三名槽位约束匹配：按官方每个第三名槽位允许的小组集合分配最佳第三名。
- 修复最佳第三名重复排名 bug：最佳第三名必须复用同一次小组排名，避免同一支队同时成为小组第二和第三。
- 新增 15 秒 API 短缓存，同一 `simulations` 参数在一个刷新窗口内复用结果，避免前端轮询和多标签重复重算。
- 前端开发环境直接请求 `127.0.0.1:8000`，生产环境仍使用相对 `/api`。

### 验证结果

- `npm run test:model` 通过，17 个测试。
- `npm run build` 通过。
- 直连 API：首次 `/api/match-prediction?simulations=1200` 约 0.9 秒，缓存命中约 0.013 秒。
- 浏览器自动化检查：桌面和手机都进入 `真实模型`，48 队冠军榜完整，五大盘面完整，页面无横向滚动。

### 当前判断

官方固定路径已经接入，模型路径可信度明显提升。仍未完成的是 Annexe C 的 495 种第三名组合完整表；当前使用的是官方槽位允许集合的确定性匹配，不会产生无效 32 强，但还不是完整 Annex C 数据表。

## 迭代记录：新闻原始数据到事件审核管线

日期：2026-06-14

### 本次目标

把新闻事件从直接写结构化 `events.json`，升级为“原始新闻输入 → 来源评级 → 事件候选 → 入模/待审核/忽略”的数据流。

### 已完成内容

- 新增 `backend/data_files/news-sources.json`，维护新闻来源和 S/A/B/C/D 等级。
- 新增 `backend/data_files/raw-news.json`，作为后续抓取器或人工录入的原始新闻输入。
- 新增 `NewsSource`、`RawNewsItem` 数据结构。
- 增加原始新闻校验：来源必须存在，引用球队必须存在，新闻 id 不能重复。
- 新增关键词规则，把原始新闻推断为事件因子、方向和强度。
- 新增事件动作：`apply` 入模、`watch` 待审核、`ignore` 忽略。
- 模型只让 `action=apply`、来源权重大于 0、且绑定具体球队的事件进入概率修正。
- `/api/events` 暴露事件列表、来源等级、审核动作和摘要。
- `event_summary` 增加 `reviewRequired`。
- 没有绑定具体球队的事件展示为“全局备注”，不再误写成“可入模型”。
- `npm run validate:data` 增加新闻来源和原始新闻统计校验。

### 验证结果

- `npm run validate:data` 通过。
- `npm run test:model` 通过，22 个测试。
- `npm run build` 通过。
- `/api/events` 返回摘要：7 条事件，3 条入模，2 条忽略，1 条待审核，4 条原始新闻。

### 当前判断

新闻层已经从静态样例推进到可运营的数据流雏形。后续外部抓取任务只需要写入 `raw-news.json` 或同结构数据源，再由后台审核决定是否入模。
