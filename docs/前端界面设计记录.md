# 前端界面设计记录

## 2026-06-14：预测工作台信息架构调整

### 本次目标

把首页从通用世界杯预测门户，调整为面向赌球用户和专业博主/分析号的每日预测工作台。

### 已完成内容

- 首屏改为“今日预测工作台”，突出今日重点比赛。
- 首页新增最可能比分 Top 3、比分矩阵、大小球 2.5、BTTS 和模型公平概率。
- 新增市场价格源状态提示，当前明确标注为待接入，不伪装真实市场价格。
- 新增今日可讲选题，服务专业博主/分析号的内容生产。
- 路径传导模块前置，展示单场赛果如何影响小组和冠军概率。
- 单场快速预览接入模型公平概率、进球市场和比分矩阵。
- 单场详情页的完整预测包调整为：比分矩阵、进球市场、模型公平概率、路径传导和博主素材。

### 关键决策

- 第一优先级不再是泛用户的冠军榜，而是单场比分分布、90 分钟胜平负、进球市场视角和路径影响。
- 赌球用户需要的是模型概率和市场价格的差值，但在真实授权价格源接入前，只展示模型公平概率和“市场价格源待接入”。
- 专业博主/分析号需要可讲述素材，所以首页保留选题卡和变化原因，而不是只给数字。
- 页面继续使用“概率分析”定位，不出现投注建议。

### 未解决问题

- 还没有接入授权市场价格源，因此无法展示真实模型-市场差值。
- 还没有球员进球、牌数、角球等 props 预测。
- 单场页完整预测仍依赖付费解锁状态，未做内容截图导出。

### 下一步

- 接入市场价格源后，补“模型概率 vs 市场价格”差值模块。
- 增加博主可截图预测卡和短文案导出。
- 补球员 props 前，先确认数据授权和合规边界。

## 2026-06-14：移动端 App 化分屏改版

### 本次目标

吸收客户反馈，把移动端从“所有模块往下拉的长页面”改成真正的 App 式分屏体验。

### 已完成内容

- 首页新增底部固定功能栏。
- 底部功能栏只保留 5 个入口：预测、赛程、榜单、新闻、解锁。
- 每个入口对应一个分屏，不再把所有模块直接堆在同一个页面。
- 主预测屏优先展示结果：重点比赛、最可能比分、胜平负概率、Top 3 比分和两条原因。
- 进球预测四项改为移动端两列紧凑排列。
- 胜平负概率移动端保持三列排列。
- 新闻和模型方法合并到“新闻”分屏，承接“为什么合理”和“用了什么数据”。
- 赛程分屏承接未开赛比赛池和单场快速预览。
- 榜单分屏承接冠军概率榜、今日概率变化和路径传导。

### 关键决策

- 主屏只回答“这场怎么看”，不承载所有解释。
- 用户先看结果，再看原因，再看方法。
- 底部导航的每个入口必须和预测业务强相关，不能为 App 感随意加入口。
- 四项以内的 KPI 卡片在移动端优先两列排列，避免单列大卡造成无效滚动。

### 未解决问题

- 当前仍是 Web App，并非原生 iOS / Android App。
- 复杂数据页仍可能有屏内滚动，但不会再把所有业务模块堆成一个长首页。
- 支付入口仍等待客户真实微信 / 支付宝接口。

### 验证与部署

- `npm run test:model` 通过，当前 114 个测试。
- `npm run build` 通过。
- `npm run deploy:web` 成功部署到 Cloudflare Pages。
- 正式地址 `https://world-cup-prediction-system.pages.dev/` 手机视口验收通过。
- 线上接口请求 `match-prediction`、`upcoming-matches`、`finished-matches`、`access-options` 均返回 200。

### 下一步

- 如果客户继续要求更像原生 App，可增加页面切换动效和单屏高度控制。
- 单场详情页后续也应按同样原则拆成“结果 / 比分 / 路径 / 解锁”几屏，而不是继续长页。

## 2026-06-14：客户二次反馈后的移动端收缩

### 本次目标

按客户反馈继续收缩移动端页面，让前台像真正 App 页面，而不是把后台运转、模型状态和长内容块暴露给用户。

### 已完成内容

- 公开首页移除“真实模型、模拟次数、入模 / 忽略、市场价格源待接入、赛果记录”等后台或运营口径。
- 顶部标题从“世界杯预测工作台”收缩为“世界杯预测”，副标题只保留更新时间，避免副标题大过主标题。
- 主预测屏把进球概率并回同一张结果卡，不再额外追加第二个大面板。
- 赛程屏只保留未开赛比赛列表，点击比赛进入单场页，不再在同屏堆快速预览。
- 榜单屏只保留冠军榜前 5 和今日变化摘要，路径传导不再占用首屏。
- 新闻屏把接口返回的后台字段翻译为用户口径，例如“入模 / 不入模型”改为“已确认 / 不采信”。
- 解锁屏移除赛果记录，只保留付费解锁入口；支付未配置时显示“支付暂未开放”，不显示客户接口配置状态。
- 移动端样式整体压缩：标题、概率数字、比分卡、新闻卡、进球概率四宫格和底部导航都降低高度。

### 关键决策

- 用户前台只回答三件事：结果是什么、为什么这样判断、怎么解锁更多。
- 后台运行状态、数据健康、事件审核、接口配置状态只属于 `/admin` 或开发文档，不进入公开首页。
- 每个底部入口的内容要尽量在 390px 手机首屏内完成，复杂解释后移到单场页或后台。

### 验证

- `npm run test:model` 通过，当前 114 个测试。
- `npm run build` 通过。
- 390×844 手机视口逐屏检查通过：预测、赛程、榜单、新闻、解锁都在底部导航上方收住。
- 手机视口检查无横向溢出。
- 前台可见文字未出现“真实模型、入模、忽略、次模拟、市场价格源待接入、World Cup Forecast Desk、模型方法、API 连接、客户接口待配置、赛果记录”等后台口径。
