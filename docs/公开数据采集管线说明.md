# 公开数据采集管线说明

日期：2026-06-15

## 目标

把公开网页、RSS 和官方 JSON 接口统一接入预测系统，形成稳定的新闻 / 伤病 / 首发 / 天气 / 训练线索流水线。

这条管线不是“看见网页就重权入模”，而是：

```text
公开网页 / RSS / 官方接口
→ 来源登记
→ 抓取适配器
→ 结构化解析
→ 去重
→ 球队 / 球员匹配
→ 来源等级判断
→ 多源交叉验证
→ 入模 / 待审 / 忽略
→ 生成解释
```

## 命令

```bash
npm run public:data:update
```

生产日更已经接入：

```bash
npm run daily:update:production
```

执行顺序是：

1. `public:data:update` 抓公开数据并写入 `raw-news.json`。
2. `run_daily_update.py` 抓赛果、重载事件、重建 50,000 次模拟快照。
3. `report:model-quality` 生成模型质量报告。

## 默认源配置

默认配置：

```text
backend/data_files/public-data-sources.json
```

当前支持三类源：

```json
{
  "id": "bbc-football-rss",
  "kind": "rss",
  "source": "bbc",
  "url": "http://newsrss.bbc.co.uk/rss/sportonline_uk_edition/football/rss.xml",
  "team": null
}
```

```json
{
  "id": "fifa-team-news-page",
  "kind": "html",
  "source": "fifa",
  "url": "https://www.fifa.com/example-team-news",
  "team": null
}
```

```json
{
  "id": "official-news-json",
  "kind": "json_news",
  "source": "fifa",
  "url": "https://example.com/news.json",
  "itemsPath": "items",
  "titleField": "headline",
  "summaryField": "summary",
  "urlField": "url",
  "publishedField": "publishedAt"
}
```

## 结构化输出

每条线索会写入：

- `team`：根据球队中文名、队码和内部 key 自动匹配。
- `players`：如果配置球员别名，会匹配球员 key。
- `category`：`injury`、`suspension`、`availability`、`lineup`、`weather`、`training`、`general`。
- `factor`：`attack`、`defense`、`goalkeeper`、`path`、`squad`。
- `direction`：`1` 利好，`-1` 利空，`0` 中性或待确认。
- `confidence`：自动解析置信度。
- `sourceRegistryId`：来源登记表中的 id。
- `kind`：`rss`、`html` 或 `json_news`。

## 分流规则

- S / A / B 来源：默认进入 `apply`，除非状态是 `rumor` 或 `unverified`。
- C 来源：单源进入 `watch`，多源交叉或人工确认后进入 `apply`。
- D 来源：进入 `ignore`。
- 同一球队、同一事件类别、同一因子、同一方向，如果来自至少两个不同来源，会升级为 `multi_source`。

## 仍然不能编造的数据

这条管线可以抓公开线索，但不会伪造：

- xG / xGA。
- 球员级事件数据。
- 完整首发数据库。
- 授权市场价格。
- 付费媒体全文。

这些数据如果客户后续提供授权接口，可以接到 `json_news` 或新增专用适配器。
