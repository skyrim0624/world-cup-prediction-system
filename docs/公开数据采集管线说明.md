# 公开数据采集管线说明

日期：2026-06-15

## 目标

把公开网页、RSS 和官方 JSON 接口统一接入预测系统，形成稳定的新闻 / 伤病 / 首发 / 天气 / 训练线索流水线。

这条管线不是“看见网页就重权入模”，而是：

```text
公开网页 / RSS / 官方接口
→ 来源登记
→ 抓取适配器
→ 结构化解析
→ 去重
→ 球队 / 球员匹配
→ 来源等级判断
→ 多源交叉验证
→ 入模 / 待审 / 忽略
→ 生成解释
```

## 命令

```bash
npm run public:data:update
```

生产日更已经接入：

```bash
npm run daily:update:production
```

执行顺序是：

1. `public:data:update` 抓公开数据并写入 `raw-news.json`。
2. `run_daily_update.py` 抓赛果、重载事件、重建 50,000 次模拟快照。
3. `report:model-quality` 生成模型质量报告。

## 默认源配置

默认配置：

```text
backend/data_files/public-data-sources.json
```

当前支持四类源：

```json
{
  "id": "bbc-football-rss",
  "kind": "rss",
  "source": "bbc",
  "url": "http://newsrss.bbc.co.uk/rss/sportonline_uk_edition/football/rss.xml",
  "team": null
}
```

```json
{
  "id": "fifa-world-cup-news-index",
  "kind": "html_index",
  "source": "fifa",
  "url": "https://www.fifa.com/en/tournaments/mens/worldcup/canadamexicousa2026/news",
  "baseUrl": "https://www.fifa.com",
  "team": null,
  "maxItems": 30
}
```

```json
{
  "id": "fifa-team-news-page",
  "kind": "html",
  "source": "fifa",
  "url": "https://www.fifa.com/example-team-news",
  "team": null
}
```

```json
{
  "id": "official-news-json",
  "kind": "json_news",
  "source": "fifa",
  "url": "https://example.com/news.json",
  "itemsPath": "items",
  "titleField": "headline",
  "summaryField": "summary",
  "urlField": "url",
  "publishedField": "publishedAt"
}
```

## 结构化输出

每条线索会写入：

- `team`：根据球队中文名、队码和内部 key 自动匹配。
- `players`：根据 `backend/data_files/player-aliases.json` 匹配球员 key。
- `category`：`injury`、`suspension`、`availability`、`lineup`、`weather`、`training`、`xg_proxy`、`player_status`、`market_proxy`、`general`。
- `factor`：`attack`、`defense`、`goalkeeper`、`path`、`squad`。
- `direction`：`1` 利好，`-1` 利空，`0` 中性或待确认。
- `confidence`：自动解析置信度。
- `sourceRegistryId`：来源登记表中的 id。
- `kind`：`rss`、`html`、`html_index` 或 `json_news`。

## 高阶代理信号

没有授权商业 xG / 球员事件 / 市场价格时，系统不伪造数值，而是把公开文本信号转成代理因子：

- `xg_proxy`：识别 `xG`、`xGA`、`expected goals`、`shot quality`、`big chances`、`预期进球`、`射门质量` 等公开文字。进攻端机会质量提升映射到 `attack +`，防守端 xGA / 被创造绝佳机会升高映射到 `defense -`。
- `player_status`：公开报道命中关键球员别名，并且属于伤病、停赛、可用性、首发、训练等状态类线索时，升级为球员级事件。
- `market_proxy`：识别公开报道里的 `odds`、`market`、`price`、`favorite`、`shorten`、`drift`、`赔率`、`市场`、`热门` 等市场热度文字，只作为 `path` 侧代理信号，不等同于授权赔率。

## 分流规则

- S / A / B 来源：默认进入 `apply`，除非状态是 `rumor` 或 `unverified`。
- C 来源：单源进入 `watch`，多源交叉或人工确认后进入 `apply`。
- D 来源：进入 `ignore`。
- 同一球队、同一事件类别、同一因子、同一方向，如果来自至少两个不同来源，会升级为 `multi_source`。

## 容错规则

每个来源独立抓取和解析。某个来源失败时：

- 报告中该来源标记为 `failed`，带 `error`。
- 其他来源继续导入。
- 总状态为 `partial_success`。
- 如果所有来源都失败，总状态为 `failed`。

## 仍然不能编造的数据

这条管线可以抓公开线索并生成代理因子，但不会伪造：

- 精确 xG / xGA 数值。
- 逐球员事件明细。
- 完整首发数据库。
- 授权市场价格。
- 付费媒体全文。

这些数据如果客户后续提供授权接口，可以接到 `json_news` 或新增专用适配器；在此之前，系统使用公开文本代理信号入模。
