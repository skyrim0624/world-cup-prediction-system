# 赛事数据导入格式

日期：2026-06-14

## 目标

用于把官方整理后的 48 队名单和 72 场小组赛赛程安全写入 `backend/data_files/teams.json` 和 `backend/data_files/fixtures.json`。

导入前必须确认数据来源，不使用未授权商业数据。

## 命令

后台入口：

```text
/admin
```

粘贴 JSON 后使用“导入赛事数据”。如果设置了 `WORLD_CUP_ADMIN_TOKEN`，需要先在后台填写 token。

如需回滚，在同一后台选择最近备份，或手动输入备份目录 ID 后使用“回滚赛事数据”。

```bash
npm run import:tournament-data -- --input /path/to/official-import.json
```

可选参数：

```bash
python3 scripts/import_tournament_data.py \
  --input /path/to/official-import.json \
  --data-dir backend/data_files \
  --backup-dir backend/data_files/backups
```

## JSON 结构

```json
{
  "source": "fifa-official",
  "retrievedAt": "2026-06-14T00:00:00Z",
  "teams": [],
  "fixtures": []
}
```

## team 字段

```json
{
  "key": "brazil",
  "name": "巴西",
  "code": "BRA",
  "group": "E",
  "elo": 1884,
  "attack": 90,
  "defense": 82,
  "goalkeeper": 85,
  "path": 76,
  "squad": 84
}
```

要求：

- 必须正好 48 支球队。
- 小组必须为 `A` 到 `L`。
- 每组必须正好 4 支球队。
- `key` 和 `code` 不能重复。

## fixture 字段

```json
{
  "home": "brazil",
  "away": "argentina",
  "stage": "小组赛 E 组",
  "kickoff": "2026-06-15T08:00:00-05:00",
  "status": "scheduled"
}
```

已结束比赛必须带比分：

```json
{
  "home": "spain",
  "away": "france",
  "stage": "小组赛 E 组",
  "kickoff": "2026-06-12T08:00:00-05:00",
  "status": "finished",
  "home_score": 2,
  "away_score": 1
}
```

进行中比赛也必须带当前比分：

```json
{
  "home": "brazil",
  "away": "argentina",
  "stage": "小组赛 E 组",
  "kickoff": "2026-06-15T08:00:00-05:00",
  "status": "live",
  "home_score": 1,
  "away_score": 0
}
```

要求：

- 必须正好 72 场小组赛。
- 每场引用的球队必须存在。
- 小组赛不能跨组。
- 同一组内同一对球队不能重复。
- `finished` 状态必须有 `home_score` 和 `away_score`。
- `live` 状态必须有当前 `home_score` 和 `away_score`。

## 安全机制

- 导入器先完整校验，再写入文件。
- 校验失败不会修改现有 `teams.json` / `fixtures.json`。
- 写入前自动备份当前文件到 `backend/data_files/backups/`。
- 回滚前也会自动备份当前文件，避免回滚操作本身造成不可逆覆盖。
- 备份目录已加入 `.gitignore`，避免误提交运行时备份。
- 后台导入成功后会重载模型，并写入后台审计日志。
- 后台回滚成功后会重载模型，并写入后台审计日志。
- 后台会展示最近完整备份，不完整备份不会进入可回滚列表。
