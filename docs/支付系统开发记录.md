## 本次目标

先把付费解锁的产品边界接进 API 和首页，明确哪些预测内容未来可以收费查看，同时避免把未接入的支付能力包装成已完成。

## 已完成内容

- 新增 `backend/access.py`，定义三种解锁范围：
  - 单场预测：胜平负、比分分布和路径传导。
  - 赛事全周期：冠军概率榜、路径变化和日更摘要。
  - 重点场次包：一组重点比赛的赛前预测和赛后路径复盘。
- 新增 `GET /api/access-options`。
- 接口返回 `paymentConfigured=false` 和 `payment_pending`，明确支付暂未接入。
- 首页新增 `付费解锁` 模块，展示三种产品范围。
- 首页展示声明：本产品提供概率分析和模型解释，不提供投注建议。
- 未新增购买按钮，避免出现假支付或误导性转化路径。
- 新增 `ACCESS_CONTENT`，把可收费内容拆成单场预测、整届概率和重点场次包三类门槛。
- 新增 `build_access_policy` 和 `GET /api/access-policy`，让前端、后台和未来支付系统读取同一套内容权限策略。
- 新增 `build_access_decision`，用于判断指定产品是否能访问指定内容。
- 当前支付未配置时，权限判断返回 `payment_pending`，不会误判为已解锁。

## 关键决策

- 当前只做“可收费范围”和“未接支付状态”，不提前做伪购买体验。
- 付费入口必须绑定概率分析边界，不使用投注、赔率、稳赚等表达。

## 2026-06-15：微信 JSAPI / Native 与支付宝接口接入

### 本次目标

按客户聊天截图确认的支付方式接入真实支付适配层：

- 微信 JSAPI：微信内网页直接调起微信支付。
- 微信 Native：非微信内扫码支付。
- 支付宝：扫码 / 网页类支付。

### 已完成内容

- 后端 `create_payment_order()` 在客户接口配置齐全时，会调用对应创建订单接口，不再只停留在 `customer_interface_ready`。
- 创建订单会向客户接口发送：
  - `orderId` / `outTradeNo`
  - `provider`
  - `paymentMethod`
  - `productKey`
  - `productName`
  - `description`
  - `amountLabel`
  - `amountCents`
  - `currency=CNY`
  - `metadata`
  - `notifyUrl`
  - `returnUrl`
- 后端会识别客户创建订单返回中的常见字段：
  - 二维码：`qrCodeUrl` / `codeUrl` / `payUrl` / `paymentUrl`
  - 微信 JSAPI 参数：`jsapiParams` / `jsApiParameters` / `wechatPayParams` / `payParams`
  - 上游订单号：`providerOrderId` / `prepayId` / `tradeNo`
- 返回字段可以在顶层，也可以包在 `data` / `result` / `payload` 里。
- 新增 `refresh_payment_order_status()`，支付等待页查询订单时会带 `sync=1`，后端会调用客户查单接口并同步状态。
- 新增 `POST /api/payments/notify/{provider}`，支持客户支付回调。
- 微信 Native 兼容客户微信支付后台已有固定回调：`POST /api/app-payment/wechat/notify`。
- 微信 Native 创建订单传给客户接口的 `notifyUrl` 固定使用 `https://zhugejunshi.com/api/app-payment/wechat/notify`。
- 当前回调验签默认采用：`X-Payment-Signature = HMAC-SHA256(raw_body, NOTIFY_SECRET)`。
- 支付状态归一化：
  - `SUCCESS` / `PAY_SUCCESS` / `TRADE_SUCCESS` / `TRADE_FINISHED` → `paid`
  - `WAIT_BUYER_PAY` / `NOTPAY` / `USERPAYING` / `PENDING` → `pending`
  - `EXPIRED` / `TIMEOUT` → `expired`
  - `FAILED` / `TRADE_CLOSED` / `CLOSED` / `CANCELLED` → `failed`
- 前端支付等待页支持微信 JSAPI 参数，微信内会调用 `WeixinJSBridge.invoke("getBrandWCPayRequest")`。
- Cloudflare Worker 已同步支付相关环境变量。
- `.env.production.example` 已补齐微信 JSAPI、微信 Native 和支付宝配置项。

### 需要客户提供

微信 JSAPI：

- 创建订单接口 URL。
- 查单接口 URL。
- 接口鉴权方式：Header 名称和值，或 Token。
- 回调验签密钥。
- 创建订单返回的 JSAPI 参数字段名。
- JSAPI 下单是否需要 `openid`，以及 openid 的获取方式。

微信 Native：

- 创建订单接口 URL。
- 查单接口 URL。
- 接口鉴权方式：Header 名称和值，或 Token。
- 回调验签密钥。
- 创建订单返回二维码字段名。

支付宝：

- 创建订单接口 URL。
- 查单接口 URL。
- 接口鉴权方式：Header 名称和值，或 Token。
- 回调验签密钥。
- 创建订单返回二维码或跳转 URL 字段名。

通用确认：

- 客户接口是否接受我们当前请求字段。
- 客户回调是否能按 `X-Payment-Signature` 做 HMAC-SHA256；如果不是，需要提供签名字段和验签算法。
- 是否需要正式回调域名白名单。

### 验证结果

- 新增支付测试覆盖：
  - 微信 Native 创建订单调用客户接口并保存二维码。

  - 微信 JSAPI 创建订单保存 `jsapiParams`。
  - 查单接口将 `SUCCESS` 同步为 `paid`。
  - 支付宝 `TRADE_SUCCESS` 同步为 `paid`。
