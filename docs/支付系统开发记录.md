# 支付系统开发记录

日期：2026-06-14

## 本次目标

先把付费解锁的产品边界接进 API 和首页，明确哪些预测内容未来可以收费查看，同时避免把未接入的支付能力包装成已完成。

## 已完成内容

- 新增 `backend/access.py`，定义三种解锁范围：
  - 单场预测：胜平负、比分分布和路径传导。
  - 赛事全周期：冠军概率榜、路径变化和日更摘要。
  - 重点场次包：一组重点比赛的赛前预测和赛后路径复盘。
- 新增 `GET /api/access-options`。
- 接口返回 `paymentConfigured=false` 和 `payment_pending`，明确支付暂未接入。
- 首页新增 `付费解锁` 模块，展示三种产品范围。
- 首页展示声明：本产品提供概率分析和模型解释，不提供投注建议。
- 未新增购买按钮，避免出现假支付或误导性转化路径。
- 新增 `ACCESS_CONTENT`，把可收费内容拆成单场预测、整届概率和重点场次包三类门槛。
- 新增 `build_access_policy` 和 `GET /api/access-policy`，让前端、后台和未来支付系统读取同一套内容权限策略。
- 新增 `build_access_decision`，用于判断指定产品是否能访问指定内容。
- 当前支付未配置时，权限判断返回 `payment_pending`，不会误判为已解锁。

## 关键决策

- 当前只做“可收费范围”和“未接支付状态”，不提前做伪购买体验。
- 付费入口必须绑定概率分析边界，不使用投注、赔率、稳赚等表达。
- 后续支付接入前，前端状态继续保持 `支付待接入`。

## 验证结果

- 新增 API 测试：`/api/access-options` 返回三种产品、`paymentConfigured=false` 和非投注声明。
- 新增前端契约测试：首页必须接入 `/api/access-options` 并渲染付费边界。
- 新增权限单元测试：单场预测包只能解锁单场内容，不能解锁整届概率。
- 新增 API 测试：`/api/access-policy` 返回内容门槛和所需产品。
- `npm run test:model` 通过，80 个测试。
- `npm run validate:data` 通过。
- `npm run build` 通过。
- Browser 插件调用失败，错误为 `browser.documentation is not a function`；已使用 Playwright + Chrome 复测。
- Playwright 复测桌面 1280px 和手机 390px：付费模块可见、接口返回 200、无错误遮罩、无横向溢出。
- 重启本地 API 后，`/api/access-policy` 返回 200。

## 2026-06-14：客户微信 / 支付宝接口适配框架

### 本次目标

按客户最新口径，支付接口使用客户提供的微信支付和支付宝支付接口。我们只搭支付框架，不申请或持有自己的微信 / 支付宝商户参数。

### 已完成内容

- 新增 `backend/payments.py`。
- 新增 `GET /api/payments/config`，返回微信支付、支付宝支付两个扫码渠道的客户接口配置状态。
- 新增 `POST /api/payments/orders`，创建本地支付订单意图。
- 新增 `GET /api/payments/orders/{order_id}`，查询本地订单状态。
- 支付渠道配置改为客户接口字段：
  - `CUSTOMER_WECHAT_PAY_CREATE_URL`
  - `CUSTOMER_WECHAT_PAY_STATUS_URL`
  - `CUSTOMER_WECHAT_PAY_NOTIFY_SECRET`
  - `CUSTOMER_ALIPAY_PAY_CREATE_URL`
  - `CUSTOMER_ALIPAY_PAY_STATUS_URL`
  - `CUSTOMER_ALIPAY_PAY_NOTIFY_SECRET`
- 当前不会生成测试二维码，不会伪造支付成功。
- 当前配置缺失时返回 `provider_config_required`。
- 即使客户接口地址配置齐全，在未确认客户返回字段协议前，也只返回 `customer_interface_ready`，不伪造二维码。
- 前端 `付费解锁` 模块新增微信支付 / 支付宝支付切换、扫码订单创建入口和订单状态提示。

### 关键决策

- 微信和支付宝不是我们自己的商户接入，而是客户接口适配。
- 我们负责产品包、订单意图、接口适配、状态展示和后续权限裁剪。
- 客户负责提供真实支付接口、真实二维码、查单结果和通知签名规则。
- 价格暂时显示 `待定价`，避免替客户自行定价。
