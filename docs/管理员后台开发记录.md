# 管理员后台开发记录

## 2026-06-14：预测运行监管

本次目标：

- 按客户反馈，在管理员后台直观看到每轮预测调用了哪些函数、哪些接口、哪些步骤没有执行。
- 给运营人员保留人工介入入口，方便回看模型链路和权重因子。

已完成内容：

- 新增 `/api/admin/prediction-run`。
- 后端按预测流程生成只读运行链路：数据输入、单场预测、整届模拟、解释输出、未接入能力。
- 每个步骤返回函数名、模块名、执行状态和原因。
- 接口状态覆盖预测读取、单场详情、快照重建、事件审核、赛果锁定等核心接口。
- 人工介入点覆盖比分写入、新闻录入、事件审核、快照重建、赛事数据导入和回滚。
- `/admin` 增加“预测运行监管”面板，桌面和手机均可读。

关键决策：

- 第一版不做伪实时函数追踪，而是基于当前数据、最近快照和日更状态生成可审计链路。
- “未接入能力”明确标为跳过，避免客户误以为已经接入官方首发、实时事件数据或授权市场价格源。
- 监管文字只出现在后台，不进入公开前台预测页。

未解决问题：

- 还没有持久化每一次预测请求的真实运行日志。
- 还没有管理员账号、角色权限和数据库化审计表。
- 生产环境写入型后台接口仍需迁移到正式数据库或 Cloudflare D1 / KV。

下一步：

- 如果客户继续要求逐次追踪，需要在预测入口加入 request id 和运行日志持久化。
- 后续可把每次日更快照的函数链路写入审计表，支持按日期回看。
