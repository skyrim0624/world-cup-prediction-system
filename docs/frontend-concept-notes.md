# 前端界面设计记录

日期：2026-06-13

## 本次目标

为世界杯动态预测系统生成第一轮 Web App 视觉概念图，帮助后续进入 React 前端实现。

用户明确纠偏：这不是包装图、产品展示板或多设备概念板，而是 Web App 本身。因此最终保留的是三张真实浏览器页面方向的截图式概念图。

用户追加要求：不止电脑端，也要方便移动端用浏览器访问。因此前端不是“先桌面再缩小”，而是响应式 Web App：桌面端负责完整数据视野，移动端负责单列滚动、触控优先和核心信息不丢失。

## 当前判断

最适合第一版响应式 Web App 的方向是：

1. 首页采用 `webapp-style-01-pitch-dashboard.png` 的结构和绿色足球场语气。
2. 付费专业感和数据密度参考 `webapp-style-02-forecast-lab.png`。
3. 单场实时预测页或比赛详情页参考 `webapp-style-03-forecast-console.png`。
4. 移动端不是桌面端裁切，必须参考三张 `mobile-style-*` 图单独设计信息顺序。

不采用的方向：

- 多手机壳展示板。
- 纯海报式概念板。
- 过度像小游戏的选择队伍界面。
- 与模型无关的商店、金币、随机按钮。

## 预测对象澄清记录

日期：2026-06-14

本轮讨论明确：用户打开网页后，不应该只看到静态夺冠热门，也不应该只看到正在进行中的比赛比分。产品核心应该是预测还没开始的比赛，并展示这些比赛会怎样改变整届世界杯走势。

已确认的预测范围：

- 单场预测：A 队对 B 队的胜 / 平 / 负概率；淘汰赛展示双方晋级概率。
- 比分预测：最可能比分 Top 3 和比分概率分布，产品内不称为“赔率”。
- 路径模拟：模拟未发生比赛对小组排名、淘汰赛路径、晋级概率和夺冠概率的影响。
- 动态因子：已经结束的比赛不再预测，而是锁定赛果，作为后续概率更新的权重因子。
- AI 分析：解释为什么某队胜率更高、为什么路径变难、为什么夺冠概率上调或下调。

对现有 Forecast Console 原型的影响：

- 当前页面强项是单场氛围和胜平负概率，但它偏“实时比赛终端”，需要调整成“未开赛单场预测终端”。
- 顶部比分条应从已发生比分，调整为对阵信息、开赛时间、比赛阶段和预测更新时间。
- 实时胜平负概率模块可以保留，但文案要改成赛前预测概率。
- 需要新增比分概率模块，用于展示最可能比分和比分分布。
- 晋级路径模拟需要从视觉符号图，调整为更明确的“如果 A 赢 / 打平 / B 赢，会影响哪些队”的传导结果。
- 首页或首屏需要同时展示“今日关键比赛预测”和“冠军概率榜”，避免用户误以为产品只预测单场。

下一步建议：

- 不马上写代码，先确认信息架构。
- 优先改 mock 数据结构，而不是先堆 UI。
- 前端下一版围绕四个核心模块重排：今日关键比赛、单场胜平负与比分概率、路径传导、整届冠军概率变化。
