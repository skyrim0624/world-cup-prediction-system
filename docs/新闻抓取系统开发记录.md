## 迭代记录：远程 Feed 空响应容错

日期：2026-06-16

### 本次目标

修复 GitHub Actions 定时日更因为第三方远程 RSS 临时空响应或坏 XML 直接失败的问题。

### 已完成内容

- `parse_news_feed` 对空 Feed 和坏 XML 抛出明确错误，不再只暴露底层 `no element found`。
- `run_daily_update` 对远程 `url` 新闻源逐源容错：单个源失败会写入 `feeds.items[].status = failed` 和错误原因，但不会阻断快照生成。
- 本地 `input` 新闻源仍保持严格失败策略，避免配置文件缺失被静默吞掉。
- 日更 CLI 输出新增 `新闻源失败`，方便在 Actions 日志里快速判断是否是第三方源波动。

### 验证结果

- 新增测试覆盖空 Feed 错误提示。
- 新增测试覆盖远程坏 Feed 失败后日更继续生成快照。
- `test_news_feed.py`、`test_daily_update.py`、`npm run validate:data`、`npm run build` 均通过。

### 当前判断

这类失败不是代码部署失败，也不是 GitHub 上不是最新版，而是第三方 RSS 在某次定时抓取时返回不可解析内容。现在远程 Feed 波动会被降级为数据源失败报告，日更主流程继续产出快照。
