# 日更流程说明

日期：2026-06-14

## 目标

把“新闻抓取、事件入模、赛果 / 进行中比分、快照重建”组织成一条可重复执行的每日更新流程。

## 日更命令

网页后台入口：

```text
/admin
```

部署时可设置后台写入 token：

```bash
WORLD_CUP_ADMIN_TOKEN=your-secret-token npm run dev:api
```

设置后，`/admin` 里的新闻录入、事件审核、比分写入和快照重建都需要填写同一个 token。

```bash
npm run daily:update -- --feed-config path/to/feed-config.json --simulations 50000 --report reports/daily-2026-06-14.json
```

不接 Feed 时，也可以只重载当前本地数据并生成快照：

```bash
npm run daily:update
```

默认会把最近一次日更状态写入：

```text
backend/data_files/daily-update-status.json
```

这份状态文件是运行产物，不进入 Git。后台 `/admin` 会读取它并显示最近日更时间、执行状态、新增新闻、模拟次数、锁定赛果和进行中比赛数量。

后台会同时给出“日更健康”：

- `正常`：最近一次成功日更仍在 24 小时内。
- `过期`：最近一次成功日更已经超过 24 小时。
- `失败`：最近一次日更命令执行失败。
- `未执行`：没有可读取的日更状态。

如果日更命令失败，CLI 会非零退出，并写入：

```json
{
  "status": "failed",
  "updatedAt": "失败时间",
  "error": "失败原因"
}
```

后台会显示“失败”，方便运营人员区分“未执行”和“执行失败”。

如需指定状态文件路径：

```bash
npm run daily:update -- --status reports/latest-daily-status.json
```

## Feed 配置格式

可复制样例见 `docs/feed-config.example.json`。

```json
[
  {
    "input": "feeds/reuters.xml",
    "source": "reuters",
    "team": "brazil"
  },
  {
    "input": "feeds/fifa.xml",
    "source": "fifa",
    "team": null
  }
]
```

说明：

- `input` 支持相对 `feed-config` 文件所在目录的路径。
- `source` 必须存在于 `backend/data_files/news-sources.json`。
- `team` 可以为空；为空时新闻先进入全局待审或备注流。
- 导入新闻默认是 `single_source`，不会绕过来源评级和审核链路。

## 比赛状态更新

进行中比分：

```bash
curl -X POST http://127.0.0.1:8000/api/fixtures/live \
  -H 'Content-Type: application/json' \
  -d '{"home":"brazil","away":"argentina","homeScore":1,"awayScore":0}'
```

已完赛比分：

```bash
curl -X POST http://127.0.0.1:8000/api/fixtures/result \
  -H 'Content-Type: application/json' \
  -d '{"home":"brazil","away":"argentina","homeScore":2,"awayScore":1}'
```

## 模型处理原则

- `scheduled`：未开赛，正常预测。
- `live`：进行中，当前比分进入模拟，后续进球按剩余比赛保守采样。
- `finished`：已完赛，比分锁定，不再预测。

## 推荐每日顺序

1. 更新进行中比分或已完赛比分。
2. 导入公开新闻 Feed。
3. 在 `/admin` 录入人工新闻线索，并确认或忽略待审事件。
4. 执行 `npm run daily:update` 生成最新快照。
5. 在 `/admin` 检查日更状态、`/api/model-status` 和首页展示。
