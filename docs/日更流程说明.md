# 日更流程说明

日期：2026-06-14

## 目标

把“新闻抓取、事件入模、赛果 / 进行中比分、快照重建”组织成一条可重复执行的每日更新流程。

## 日更命令

网页后台入口：

```text
/admin
```

```bash
npm run daily:update -- --feed-config path/to/feed-config.json --simulations 50000 --report reports/daily-2026-06-14.json
```

不接 Feed 时，也可以只重载当前本地数据并生成快照：

```bash
npm run daily:update
```

## Feed 配置格式

可复制样例见 `docs/feed-config.example.json`。

```json
[
  {
    "input": "feeds/reuters.xml",
    "source": "reuters",
    "team": "brazil"
  },
  {
    "input": "feeds/fifa.xml",
    "source": "fifa",
    "team": null
  }
]
```

说明：

- `input` 支持相对 `feed-config` 文件所在目录的路径。
- `source` 必须存在于 `backend/data_files/news-sources.json`。
- `team` 可以为空；为空时新闻先进入全局待审或备注流。
- 导入新闻默认是 `single_source`，不会绕过来源评级和审核链路。

## 比赛状态更新

进行中比分：

```bash
curl -X POST http://127.0.0.1:8000/api/fixtures/live \
  -H 'Content-Type: application/json' \
  -d '{"home":"brazil","away":"argentina","homeScore":1,"awayScore":0}'
```

已完赛比分：

```bash
curl -X POST http://127.0.0.1:8000/api/fixtures/result \
  -H 'Content-Type: application/json' \
  -d '{"home":"brazil","away":"argentina","homeScore":2,"awayScore":1}'
```

## 模型处理原则

- `scheduled`：未开赛，正常预测。
- `live`：进行中，当前比分进入模拟，后续进球按剩余比赛保守采样。
- `finished`：已完赛，比分锁定，不再预测。

## 推荐每日顺序

1. 更新进行中比分或已完赛比分。
2. 导入公开新闻 Feed。
3. 在 `/admin` 事件审核台确认或忽略待审事件。
4. 执行 `npm run daily:update` 生成最新快照。
5. 检查 `/api/model-status` 和首页展示。
