# 日更流程说明

日期：2026-06-14

## 目标

把“新闻抓取、事件入模、赛果 / 进行中比分、快照重建”组织成一条可重复执行的每日更新流程。

## 日更命令

网页后台入口：

```text
/admin
```

部署时可设置后台 token：

```bash
WORLD_CUP_ADMIN_TOKEN=your-secret-token npm run dev:api
```

设置后，`/admin` 的数据读取、新闻录入、事件审核、比分写入和快照重建都需要填写同一个 token。

```bash
npm run daily:update
```

默认会读取：

```text
backend/data_files/daily-feed-sources.json
backend/data_files/score-sources.json
```

需要临时指定其它 Feed 配置时：

```bash
npm run daily:update -- --feed-config path/to/feed-config.json --score-config path/to/score-config.json --simulations 50000 --report reports/daily-2026-06-14.json
```

不接 Feed、只重载当前本地数据并生成快照时，可以传空配置或直接调用脚本不带 `--feed-config`。

检查日更是否仍然健康：

```bash
npm run daily:check
```

这个命令适合接入 cron、GitHub Actions、服务器监控或其它外部告警系统。状态为 `正常` 时退出码为 0；`未执行`、`过期`、`失败` 时退出码为 1。

默认会把最近一次日更状态写入：

```text
backend/data_files/daily-update-status.json
```

这份状态文件是运行产物，不进入 Git。后台 `/admin` 会读取它并显示最近日更时间、执行状态、新增新闻、模拟次数、锁定赛果和进行中比赛数量。

后台会同时给出“日更健康”：

- `正常`：最近一次成功日更仍在 24 小时内。
- `过期`：最近一次成功日更已经超过 24 小时。
- `失败`：最近一次日更命令执行失败。
- `未执行`：没有可读取的日更状态。

如果日更命令失败，CLI 会非零退出，并写入：

```json
{
  "status": "failed",
  "updatedAt": "失败时间",
  "error": "失败原因"
}
```

后台会显示“失败”，方便运营人员区分“未执行”和“执行失败”。

如需指定状态文件路径：

```bash
npm run daily:update -- --status reports/latest-daily-status.json
```

## Feed 配置格式

可复制样例见 `docs/feed-config.example.json`。

```json
[
  {
    "input": "feeds/reuters.xml",
    "source": "reuters",
    "team": "brazil"
  },
  {
    "url": "https://www.espn.com/espn/rss/soccer/news",
    "source": "espn",
    "team": null
  }
]
```

说明：

- `input` 支持相对 `feed-config` 文件所在目录的路径。
- `url` 支持远程 RSS / Atom 地址。
- `input` 和 `url` 二选一。
- `source` 必须存在于 `backend/data_files/news-sources.json`。
- `team` 可以为空；为空时新闻先进入全局待审或备注流。
- 导入新闻默认是 `single_source`，不会绕过来源评级和审核链路。

## 赛果源配置格式

默认赛果源配置：

```text
backend/data_files/score-sources.json
```

当前默认使用 FIFA 官方公开日历接口：

```json
[
  {
    "url": "https://api.fifa.com/api/v3/calendar/matches?locale=en&IdCompetition=17&IdSeason=285023&count=200",
    "source": "fifa",
    "format": "fifa_calendar_matches"
  }
]
```

说明：

- `format` 当前支持 `fifa_calendar_matches` 和历史兼容的 `espn_scoreboard`。
- `fifa_calendar_matches` 读取 FIFA 官方公开赛程日历，按 `MatchNumber`、球队缩写和比赛状态更新本地赛程。
- 外部队伍缩写会按 `teams.json` 里的 `code` 映射到内部球队 key。
- 外部状态为 `finished` 时锁定完赛比分。
- 外部状态为 `live` 时更新进行中比分。
- 外部状态为 `scheduled` 时跳过，不会把未开赛比赛误写成赛果。
- 小组积分由已锁定的官方赛果在模型内计算，不单独写入静态积分表。
- FIFA 日历接口当前可观察到部分官方元数据，例如阵型、场馆和天气；如果接口未提供红黄牌或完整首发，日更状态会保留缺口，运营人员通过后台补录或等待后续专项接口接入。

日更报告里的 `scores` 会包含：

- `lineupsObserved`：官方返回中观察到阵型/首发相关字段的比赛数。
- `disciplineObserved`：官方返回中观察到红黄牌/纪律字段的比赛数。
- `weatherObserved`：官方返回中观察到天气字段的比赛数。
- `stadiumsObserved`：官方返回中观察到场馆字段的比赛数。
- `standingsSource`：当前积分来源，FIFA 官方赛果更新成功时为 `computed_from_official_fifa_results`。

日更报告里的 `newsVerification` 会包含：

- `singleSource`：单源新闻线索数。
- `multiSource`：多源交叉确认线索数。
- `confirmed`：人工确认线索数。
- `reviewRequired`：仍需后台审核的线索数。
- `applied` / `ignored`：已入模和已忽略事件数。

## 比赛状态更新

进行中比分：

```bash
curl -X POST http://127.0.0.1:8000/api/fixtures/live \
  -H 'Content-Type: application/json' \
  -d '{"home":"brazil","away":"argentina","homeScore":1,"awayScore":0}'
```

已完赛比分：

```bash
curl -X POST http://127.0.0.1:8000/api/fixtures/result \
  -H 'Content-Type: application/json' \
  -d '{"home":"brazil","away":"argentina","homeScore":2,"awayScore":1}'
```

## 模型处理原则

- `scheduled`：未开赛，正常预测。
- `live`：进行中，当前比分进入模拟，后续进球按剩余比赛保守采样。
- `finished`：已完赛，比分锁定，不再预测。

## 推荐每日顺序

1. 执行 `npm run daily:update`，自动拉取赛果源和公开新闻 Feed。
2. 如外部源缺漏，再通过 `/admin` 或 API 手动补进行中比分 / 已完赛比分。
3. 在 `/admin` 录入人工新闻线索，并确认或忽略待审事件。
4. 必要时再次执行 `npm run daily:update` 生成最新快照。
5. 在 `/admin` 检查日更状态、`/api/model-status` 和首页展示。

## 部署定时任务

仓库已加入 GitHub Actions：

```text
.github/workflows/daily-update.yml
```

触发方式：

- 每 3 小时执行一次。
- 支持在 GitHub Actions 页面手动 `workflow_dispatch`。

执行内容：

1. `npm ci`
2. `npm run validate:data`
3. `npm run daily:update`
4. `npm run daily:check`
5. 如果赛程、新闻或快照有变化，自动提交 `chore: daily prediction update`
