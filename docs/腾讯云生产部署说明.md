# 腾讯云生产部署说明

日期：2026-06-15

## 目标

把预测系统从 Cloudflare 测试形态迁到腾讯云服务器长期运行。生产目标不是人工改代码后再部署，而是服务器自己定时拉取公开赛果源、新闻 Feed、重建预测快照，并暴露健康检查。

## 推荐方式：Docker Compose

1. 复制环境变量样例：

```bash
cp .env.production.example .env.production
```

2. 把 `WORLD_CUP_ADMIN_TOKEN` 改成足够长的随机值。

3. 构建并启动：

```bash
docker compose -f docker-compose.tencent.yml up -d --build
```

4. 首次生成生产快照和模型质量报告：

```bash
docker compose -f docker-compose.tencent.yml exec world-cup-prediction npm run daily:update:production
```

5. 检查生产状态：

```bash
docker compose -f docker-compose.tencent.yml exec world-cup-prediction npm run production:check
```

## 非 Docker 方式

项目默认安装到：

```text
/opt/world-cup-prediction-system/current
```

安装依赖并构建前端：

```bash
npm ci
npm run build
python3 -m pip install "fastapi>=0.128.0" "uvicorn>=0.40.0"
```

复制 systemd 文件：

```bash
sudo cp deploy/tencent-cloud/world-cup-prediction-api.service /etc/systemd/system/
sudo cp deploy/tencent-cloud/world-cup-prediction-daily.service /etc/systemd/system/
sudo cp deploy/tencent-cloud/world-cup-prediction-daily.timer /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now world-cup-prediction-api.service
sudo systemctl enable --now world-cup-prediction-daily.timer
```

复制 Nginx 配置：

```bash
sudo cp deploy/tencent-cloud/nginx.conf /etc/nginx/conf.d/world-cup-prediction.conf
sudo nginx -t
sudo systemctl reload nginx
```

## 生产运行命令

启动 API 和静态前端：

```bash
npm run production:serve
```

执行完整日更：

```bash
npm run daily:update:production
```

生产健康检查：

```bash
npm run production:check
```

生产检查会同时看：

- 日更状态是否在 2 小时内成功。
- 预测快照是否存在且在 2 小时内生成。
- 模型质量报告是否存在，Poisson 比分模型回测是否达标。
- 核心数据是否覆盖 48 队、72 场小组赛、无占位球队。

## 自动更新逻辑

`world-cup-prediction-daily.timer` 每 30 分钟触发一次：

```text
npm run daily:update:production && npm run production:check
```

日更流程会：

1. 拉取 `backend/data_files/score-sources.json` 中的 FIFA 官方公开赛果源。
2. 把外部状态为 finished 的比赛写成本地已完赛比分。
3. 把外部状态为 live 的比赛写成本地进行中比分。
4. 跳过 scheduled，不会把未开赛比赛误写成赛果。
5. 拉取 `backend/data_files/daily-feed-sources.json` 中的公开新闻 Feed。
6. 自动识别新闻涉及球队、事件类别、方向和置信度。
7. 按来源等级进入入模、待审或忽略。
8. 重新生成 50,000 次模拟快照。
9. 生成模型质量报告。
10. 写入 `backend/data_files/daily-update-status.json`。

## 可靠性边界

- Cloudflare 只作为测试站，不再作为客户生产架构依赖。
- 文件写入已经改为原子替换，避免定时任务中途失败留下坏 JSON。
- 日更有进程锁，重叠执行会跳过第二次，不会互相覆盖。
- 已完赛结果不会被较旧的 live 源回滚。
- xG、授权市场、完整首发和球员级状态不编造；系统已保留接入字段、模型函数和质量门禁，客户提供授权数据后才会实际激活。
